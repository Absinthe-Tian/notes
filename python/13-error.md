当检测到一个错误时，解释器就⽆法继续执行了，反而出现了一些错误的提示，这就是所谓的"异常"。

# 异常

```python
try:
  可能发送异常的代码
  ...
except (异常类型, ...) as e:
  出现该类型异常后处理的代码
  异常类型可以同时捕获多个
else:
  没有异常时执行的代码
  可以没有该部分语句
finally:
  无论有没有异常都会执行的代码
  可以没有该部分语句
```

> Exception是所有程序异常类的父类，指定该异常类型可以捕获所有异常，但是不够精确。

# 自定义异常类型

```python
 # 自定义异常类
class 异常类名(Exception):

	# 设置抛出异常的描述信息
```
